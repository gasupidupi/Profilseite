Block 2

In diesem Block beasste ich mich zuerst mit der Responsive Übung. Anschliessend erhielt ich nachfolgende Theorie.
  • Ein «flatdesign» ist ein minimalistisches Design.
  • Ein «onepager» ist eine Webseite ohne mehreren Seiten. Der gesamte Inhalt ist auf einer Seite ersichtlich.
  • Form follows function bedeutet, dass die Form durch die Funktion definiert wird.
  • Mobile first, lautet ein Sprichwort. Es besagt, dass man zuerst an der mobilen Version arbei-ten sollte, damit man nicht reduzieren, sondern ergänzen kann, da man dann den Platz ideal gespart hat.
  • Es gibt vier Schritte, um eine Webseite responsive zu machen.

RGB kann Werte zwischen 0 und 255 pro Farbe beinhalten. Dies resultiert zu 16.8 Möglichkeiten.
Hexadezimale Werte sehen folgendermassen aus: #FFFFFF
Es können Werte zwischen 0-9 und A-F eingegeben werden.
Folgende Bildformate sind für das Web sinnvoll.

JPG

  • besitzt 16.7 Mio Farben bzw. 24 Bit.
  • verliert Qualität bei Komprimierung.
  • besitzt keine Transparenz.
  • ermöglicht Farbverläufe.

GIF

  • ermöglicht keine Farbverläufe.
  • ermöglicht Animationen.
  • besitzt Transparenz.
  • besitzt 256 Farben bzw. 8 Bit.

PNG

  • besitzt Transparenz.
  • ist möglich in 8 Bit oder 24 Bit.
  • ermöglicht verlustfreie Komprimierung.

SVG

  • ermöglicht verlustfreie Komprimierung.
  • beinhaltet Transparenz.
  • ermöglicht Animation.
  • ermöglicht verlustfreies Zoomen.
  • kann als Code eingefügt werden.

Es gibt folgende Standardauflösungen

  • Smartphones: 320px bis 480px
  • Tablets: 768px bis 1024px
  • Computer-Desktop: 1024px+

Webseite responsive machen

  1. Den Meta-tag gilt es einzubauen. Dieser lautet folgendermassen.
    2. <meta name ="viewport" content="width=device-width, inital-scale=1">
  2. Danach sollte man die Navigation anpassen. Man darf auch keine Tabelle verwenden, da Blindenprogramme dies nicht korrekt interpretiert.
  3. Anschliessend sollte man den stylesheet anpassen und «@media {}» implementieren.
  4. Schlussendlich gilt es noch die Bilder und Videos responsive zu machen. Beispielsweise mit diesem CSS-Code.
    bild {max-width: 100%; height:auto;}

Fonts

Anhand der CSS-Regel @font-face ist es ermöglicht Fonts einzubinden. Fonts selbst befinden sich meist auf dem Server, auf welchem die Webseite gehostet ist und können potentiell heruntergeladen werden.
Woff ist ein modernes Schrift-Format.
Eine Webseite auf dem verschiedenen Font angeboten werden ist www.fontsquirrel.com.
Es gibt folgende Schriftformate:
  • TrueType Fonts (TTF)
    • Dies ist ein üppiges Format für Windows und Mac.
  • The Web Open Font Format (WOFF)
    • Dies wird oft auf Webseiten verwendet.
  • The Web Open Font Format (WOFF 2.0)
    • Dies bietet besser Kompression verglichen zu WOFF 1.0.
  • Embedded OpenType Fonts (EOT)
    • Dies biete eine kompakte Form der OpenType Fonts.
Nachfolgend ist ein Beispiel, wie man Fonts einfügt:
@font-face {
font-family: "opensansregular";
src: url('..fonts/OpenSans-Regular.ttf') format('ttf');
font-weight: normal;
font-style: normal;
}
html {
font-family: "opensansregular", sans-serif;
scroll-behavior: smooth;
}

Lightbox

Responsiveslider

Um den «Responsiveslider» im HTML zu verwenden dient folgender Code.

Dateien velinken

<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.8.3/jquery.min.js"></script>
<script src="responsiveslides.min.js"></script>

Markup hinzufügen

<ul class="rslides">
<li><img src="../img/1.jpg" alt=""></li>
<li><img src="../img/2.jpg" alt=""></li>
<li><img src="../img/3.jpg" alt=""></li>
</ul>

CSS hinzufügen

.rslides {
position: relative;
list-style: none;
overflow: hidden;
width: 100%;
padding: 0;
margin: 0;
}

.rslides li {
-webkit-backface-visibility: hidden;
position: absolute;
display: none;
width: 100%;
left: 0;
top: 0;
}

.rslides li:first-child {
position: relative;
display: block;
float: left;
}

.rslides img {
display: block;
height: auto;
float: left;
width: 100%;
border: 0;
}

JS für den Slider einbauen

<script>
$(function() {
$(".rslides").responsiveSlides();
});
</script>

Slider Einstellungen optimieren

$(".rslides").responsiveSlides({
auto: true,
speed: 500,
timeout: 4000,
pager: false,
nav: false,
random: false,
pause: false,
pauseControls: true,
prevText: "Previous",
nextText: "Next",
maxwidth: "",
navContainer: "",
manualControls: "",
namespace: "rslides",
before: function(){},
after: function(){}
});

Anschliessend erfuhr ich über den Video- und Audio-tag.

Den Code um Audio einzufügen geht wie folgt
<audio src="../audio/music.opus" type="audio/opus" con-trols></audio>

Mögliche Attribute für Audio sind:
  • autoplay
  • controls
  • loop
  • muted
  • preload (auto, none und metadata)
  • src
Den Code um Audio einzufügen geht wie folgt
<video controls><source src="../video/movie.ogv"></video>

Mögliche Attribute für Video sind:
  • autoplay
  • buffered
  • controls
  • loop
  • muted
  • played
  • preload (auto, none und metadata)
  • src
  • volume
